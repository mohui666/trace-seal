## Issues

### Squirrel lifecycle and shortcuts

- Issue: the pre-fix installed build did not handle Squirrel lifecycle arguments; an uninstall attempt exited `-1`, and installation did not reliably create shortcuts.
- Severity: release blocking for the Windows installer.
- Fix: added early Squirrel install/update/uninstall/obsolete handling, explicit shortcut creation/removal through `Update.exe`, and 12 unit tests.
- Retest: **PASS**; install created two shortcuts, uninstall exited 0 and removed the app stub/shortcuts, and reinstall restored them.

### Release label

- Issue: the renderer displayed `v0.2.0 RC`.
- Severity: release blocking presentation mismatch.
- Fix: changed the label to `v0.2.0`.
- Retest: **PASS** in the installed application.

### Windows UI automation

- Issue: the local Computer Use runtime failed to load because the bundled `@oai/sky` package did not export a required subpath.
- Severity: validation tooling limitation.
- Fix: none in the TraceSeal repository. Installation and renderer behavior were validated through installer exit codes, filesystem/process inspection, Electron CDP, DOM text, and the exposed Electron bridge.
- Retest: native Start Menu click and directory-picker interaction remain **not executed**.

### Clean Windows environment provisioning

- Windows Sandbox could not be enabled because the host component store is corrupt. `DISM /Online /Cleanup-Image /RestoreHealth` reached 100% and failed with `0x800f0915` (`The repair content could not be found anywhere`).
- Hyper-V fallback: **PASS**. `TraceSeal-Clean-Validation` completed clean-machine dependency, install, launch, rendered UI, bundled-Core, workspace, settings-recovery, uninstall, and reinstall checks.
- Automation limitation: the host Computer Use runtime could not load an exported `@oai/sky` subpath. GUI assertions therefore used Electron CDP inside the interactive VM session. This proved rendered content and runtime responses but did not click the native folder picker or observe SmartScreen.
- Release impact: no release-blocking failure was found. Release notes must retain the Windows x64 scope, unobserved SmartScreen behavior, native-picker automation limitation, and no macOS/Linux validation.
