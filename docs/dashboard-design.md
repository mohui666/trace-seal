# TraceSeal Dashboard 页面结构设计

## 文档状态

本文是 Dashboard 的设计草案。当前 MVP 尚未实现 `dashboard/` 前端或后端 API；当前可用的查看方式是 `python -m traceseal replay runs/latest` 和 `python -m traceseal explain runs/latest`。


## 1. 页面概览

Dashboard 是 TraceSeal 的可视化界面，用于展示 Agent 执行轨迹、风险分析、事故回放等信息。

## 3. 页面交互设计

### 3.1 导航

- 顶部导航栏固定，支持快速切换页面
- 当前页面高亮显示
- 支持键盘快捷键（1-9 切换页面）

### 3.2 筛选和搜索

- 时间范围筛选（今天/最近7天/最近30天/自定义）
- 操作类型筛选（file/shell/http/git）
- 风险等级筛选（low/medium/high/critical）
- 关键词搜索

### 3.3 详情展开

- 点击事件可展开查看详细信息
- 支持折叠/展开所有事件
- 支持标记重要事件

### 3.4 导出功能

- 导出轨迹为 JSON 文件
- 导出报告为 PDF
- 导出日志为文本文件

## 4. 响应式设计

### 4.1 桌面端（>= 1024px）

- 完整布局，所有功能可见
- 侧边栏显示导航
- 多列布局展示数据

### 4.2 平板端（768px - 1023px）

- 折叠侧边栏
- 双列布局
- 触摸友好的按钮大小

### 4.3 移动端（< 768px）

- 单列布局
- 汉堡菜单导航
- 卡片式展示事件
- 简化详情页面

## 5. 技术实现建议

### 5.1 前端技术栈

- **框架**: React 或 Vue 3
- **UI 组件库**: Ant Design 或 Element Plus
- **图表**: ECharts 或 D3.js
- **状态管理**: Redux 或 Pinia
- **路由**: React Router 或 Vue Router

### 5.2 后端 API

```
GET /api/runs              # 获取运行列表
GET /api/runs/:id          # 获取运行详情
GET /api/runs/:id/events   # 获取事件列表
GET /api/runs/:id/diff     # 获取文件 diff
GET /api/runs/:id/shell    # 获取 shell 输出
GET /api/runs/:id/http     # 获取 HTTP 请求记录
GET /api/runs/:id/git      # 获取 Git diff
GET /api/runs/:id/analysis # 获取事故分析
POST /api/runs/:id/replay  # 回放运行
```
